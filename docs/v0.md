## Goal of v0

Validate the core: a Thai retail investor who trades US stocks via Dime can get their portfolio in, see a useful dashboard, and chat with an AI agent about it.

---

## Scope decisions (decided 2026-05-01)

- **Asset class:** US stocks only. SET, crypto, mutual funds — post-v0. *Why US:* most Thai retail using fractional-stock platforms like Dime go to US stocks because returns are more attractive than SET. That's where the v0 user is.
- **Ingestion (three paths):**
  1. **Dime slip** — image generated by Dime after each transaction, saved to the user's phone photo gallery. Mechanism for getting that image into our app: TBD.
  2. **Manual entry**
  3. **General vision-LLM extraction** — catch-all for any other broker's slip / screenshot
- **Surfaces:**
  1. **Dashboard / visualization / analytics** — the presentation layer (Home + Position Detail; Performance tab deferred to v0.5). See §Dashboard area design below.
  2. **AI chat agent** — reactive only, with tool access to stock data + user-scoped portfolio + general context. Runs on user query (chatbot for now). *No proactive push, no daily email digest in v0.*
- **UI language:** English. (US data sources are English-native; Thai UI is post-v0.)
- **Target user:** unchanged from [target_user.md](target_user.md) — Thai retail investor. The v0 wedge is the slice that holds US stocks (via Dime / Webull TH / Liberator).

## Dashboard area design (decided 2026-05-01)

### Surfaces in v0

- **Home** (always-on screen)
- **Position detail** (drill-in from any holding)
- **Performance tab** — deferred to v0.5. Contains the strongest "guide thinking" features (TWR/MWR side-by-side, counterfactual benchmark, FX-decomposed P&L). Pull forward later.

### External data the dashboard needs

- Current price + previous close per holding
- 1Y daily OHLCV per holding (for portfolio chart)
- USD↔THB rate (current + history)
- Basic fundamentals per holding: market cap, P/E, EPS, revenue, 52-week high/low
- Recent news (last 7 days, per-ticker)
- Next earnings date
- Company name + logo

### Computed internally (not external)

- Cost basis, P&L, lots — from our portfolio store + current prices
- Total portfolio value (THB and USD)
- Composition %, concentration

### Free API stack

- **Finnhub free** — primary. Real-time US quotes (60 req/min), historical OHLCV, per-ticker news, earnings calendar, basic fundamentals
- **FMP free** — supplementary. Deeper fundamentals (250 req/day), dividends, splits
- **Frankfurter** (or ExchangeRate-API) — USD↔THB, unlimited free basic
- Cache aggressively — fundamentals once/day per ticker, news once/hour, prices with 5-min TTL

Free-tier headroom: ~100+ unique tickers across early users. Architecture details for caching strategy in [architecture.md](architecture.md).

### Indicators in v0 dashboard

Near-zero. Indicators are mostly noise for novice retail; "guides thinking" comes from framing (composition, FX decomposition, % from high), not from RSI/MACD on the home screen.

Two included:
- **SMA 50 + SMA 200** overlay on the price chart — industry-standard trend context, optional toggle
- **% from 52-week high** — simple drawdown framing (not really an indicator, just derived data)

Other indicators (RSI, MACD, Bollinger, ATR, etc.): not on the dashboard — chat agent computes them on demand when the user asks.

### Filings on the dashboard

Not surfaced. Chat agent uses filings as RAG context; the dashboard doesn't show them directly.
